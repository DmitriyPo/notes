ECMAScript 2017

Вот ссыль на спецификацию (самая свеженькая на даный момент - 8-ое издание), которую я рассмотрю здесь. Начнем с краткого обзора того, что такое ECMAScript (он же JavaScript).

Краткий обзор

ECMAScript это объектно-ориентированный язык программирования для выполнения вычислений и управления вычислительными объектами внутри среды выполнения. Изначально он был разработан как язык скриптов, при этом стал широко используемым как стандартный язык программирования.

Веб-браузер предоставляет окружение ECMAScript для вычислений на стороне клиента (это объекты, что представляют окна, меню, диалоговые окна, фреймы, историю, куки, ввод/вывод и др.; и способы, чтобы прикрепить код к событиям аля мыши, отправки формы, загрузки страницы, ошибок, выбора и тд.). Веб-сервер предоставляет другое окружение для вычислений на стороне сервера (а это объекты, которые представляют запросы, клиентов и файлы; механизмы блокировки и обмена данными). Используя совместное выполнение скриптов на стороне браузера (клиента) и сервера, становится возможным распределить вычисление между ними (клиентом и сервером), предоставляя индивидуальный пользовательский интерфейс для веб-приложения. Каждый браузер и сервер вместе завершают окружение выполнения ECMAScript.

Перед тем, как дальше углубиться в дебри языка, рассмотрим в сжатом виде ECMAScript. Его основой являются объекты: они обеспечивают базу языка и его основные возможности, и программа ECMAScript это набор взаимодействующих объектов. А объект это собрание свойств, у которых есть атрибуты. Свойства же в свою очередь это контейнеры для других объектов, примитивных значений, или функций. Примитивным есть значение, что представляет один из следующих заложенных типов: Undefined (неопределенное), Null (значение для нихуя), Boolean (true/false), Number (число), String (строка), и Symbol (символ); объект это представитель типа Object; а функция это вызываемый объект. И функция, связанная с объектом через свойство, зовется методом.

Набор ECMAScript объектов (что являют его сущность) включает в себя глобальный объект; объекты, что есть основными для динамической семантики (формальной модели) языка (Object, Function, Boolean, Symbol, и Error); объекты, что представляют и манипулируют числовыми значениями (Math, Number, и Date); объекты, обрабатывающие текст (String и RegExp); объекты, которые есть пронумерованными наборами значений (Array и другие его виды); ключеве наборы (Map и Set); объекты, что поддерживают структурированные данные (JSON, ArrayBuffer, SharedArrayBuffer, и DataView); объекты, что поддерживают абстрактные понятия управления (функции-генераторы и Promise); отражающие объекты (Proxy и Reflect).

Еще объекты определяют набор операторов (унарные, запятые, логические, назначения, равенства, относительные и тд). Также большие програмы ECMAScript поддерживают модульность, что позволяет разделить програму на множество последовательностей конструкций и объявлений. Модуль определяет, какие объявления (которые он использует) предоставляются другими модулями, и какие из его объявлений доступны для использования другими модулями. Да и синтаксис языка лайтовый (нет, например, строгой типизации), хоть в нем и существует возможность использовать строгий режим.

Объекты

Объекты ECMAScript не основаны на классах, хоть и имеет в себе синтаксис для их определения. Объекты могут быть созданы разными способами, включая возможность их создания через литеральное описание или через конструкторы (объекты от конструкторов инициализируются через назначение их свойствам начальных значений). Конструктор является функцией, что имеет свойство 'prototype', которое используется, чтобы реализовать наследование, основанное на прототипировании и разделяемых свойствах.

Каждый объект, созданный с помощью конструктора, имеет неявную ссылку (что зовется прототипом объекта) на значение свойства 'prototype' его конструктора. Более того, прототип может иметь неявную ссылку на свой прототип и тд; это называется цепочкой прототипов. Когда в объекте делается ссылка на свойство, она ссылается на свойство с таким именем в первом (попавшемся в цепочке) объекте, в котором оно содержится (имеется в виду, что будут рассматриваться объекты в цепочке прототипов до тех пор, пока не будет найден именно тот первый объект, содержащий данное искомое свойство, на которое ссылаются).

В ECMAScript состояние и методы переносятся объектами, и в то же время структура, поведение и состояние наследуются. Объекты, которые не содержат в себе конкретное свойство, что содержится в их прототипе, разделяют это свойство и его значение (через наследование).

Строгий вариант ECMAScript

ECMAScript определяет строгий вариант языка. Он исключает некоторые определенные синтаксические и семантические свойства обычного ECMAScript и меняет детализированную семантику некоторых его свойств. Строгий вариант также определяет дополнительные условия для ошибок, которые должны быть зарегистрированы путем выброса исключения об ошибке в ситуациях, которые не определены как ошибочные в не строгой форме языка.

Строгий вариант языка обычно указывается как строгий режим (strict mode). Строгий режим налагает ограничения, что имеют только локальный эфект внутри синтаксической единицы ECMAScript (на индивидуальном уровне для этой единицы). Програма может состоять из синтаксических единиц ECMAScript, что написаны в строгом и не строгом режиме.

Типы данных и значений в ECMAScript

Типы языка ECMAScript

Типы языка есть следующие: Undefined (неопределенный), Null (нулевой), Boolean (для булевских значений), String (для значений в виде строки), Symbol (для символов), Number (для числовый значений), и Object (для объектов). Кратко рассмотрим каждый (дадим обозначение).

Неопределенный тип (Undefined)

Этот тип имеет только одно значение - undefined. Каждая переменная, которой не было назначено какое-либо значение (кроме того, что соотвествует данному типу), будет иметь в качестве значения undefined.

Нулевой тип (Null)

Этот тип имеет только одно значение - null.

Булев тип (Boolean)

Булев тип (логический) служит для представления логической сущности, что имеет два значения: true и false.

Строковый тип (String)

Строковый тип это набор упорядоченных последовательностей из элементов, которые имеют нулевое или более 16-битное целочисленное значение без знака, вплоть до максимального их числа, что есть 253-1. Данный тип в основном используется, чтобы представить текстовую информацию в работающей програме ECMAScript, и в этом случае каждый элемент обрабатывается как UTF-16 значение единицы кода (UTF-16 это кодировка символов Юникода, способная к шифрованию всех 1,112,064 корректных кодовых точек, а они в свою очередь представляют целочисленные значения, что образовывают кодовое пространство для кодировки этих символов в диапазоне от 0 и до 10FFFF16). Каждый элемент рассматривается как занимающий позицию внутри последовательности. А эти позиции, в свою очередь, индексируются не отрицательными целыми числами. Первый элемент будет иметь индекс 0, второй - 1, и тд. Длина строки это число элементов (16-битных значений) внутри нее (пустая строка имеет длину 0).

Там, где операции ECMAScript интерпретируют строковые значения, каждый элемент интерпретируется как отдельная кодовая точка UTF-16. ECMAScript не накладывает никаких ограничений или требований на последовательность кодовых точек в значении строки, хоть это и может призвести к каке (к их плохому формированию). Операции, что не интрпретируют контент строки, обрабатывают его, как универсальные (нерасчлененные) 16-битные целочисленные переменные без знака.

Тип для символов (Symbol)

Этот тип является набором всех не строковых значений, которые могут быть использованы, как ключевое выражение свойства объекта. Каждое возможное значение этого типа уникальное и неизменное. Каждое значение символьного типа неизменно содержит соотвествующее значение, что зовется [[Description]], которое или undefined, или строковое значение.

Известные символы, которые используются в качестве ключевых выражений свойств, чьи значение служат как точки расширения алгоритма спецификации, наводятся в этой таблице. Транслэйтом этого я займусь, когда будет необходимость (сейчас ее нет).

Числовой тип (Number)

Данный тип представляет значения, которые являются 64-битными формата IEEE 754-2008, и их количество составляет 18437736874454810627 (264-253+3). Дохера, в общем. Среди них есть особенные. Первым особенным значением есть NaN (“Not-a-Number”, которое производится программным выражением NaN). В коде ECMAScript NaN значения не отличимы друг от друга, при этом в некоторых реализациях код способен обнаружить разницу между разными NaN значениями. Разные значения NaN не равны друг другу.

Еще одними особенными есть значения позитивной и отрицательной беснонечности, которые изображают с помощью символов +∞ и -∞ (оба они производятся программными выражениями +Infinity (или просто Infinity) и -Infinity).

Остальные значения являются конечными, и половина из них позитивные значения, а другая - отрицательные. Для каждого конечного числового значения существует соответствющее отрицательное одинаковой величины по модулю.

В ECMAScript не один ноль, а два: позитивный и отрицательный. Выражаются они символами +0 и -0 (два этих числовых значения, выражающие ноль, производятся программными выражениями +0 (или просто 0) и -0).

Все положительные и отрицательные целые числа, величина которых по модулю не превышает 253, представлены в типе чисел.
